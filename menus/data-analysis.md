# 数据分析套餐
> Data大厨 + Code大厨 + Doc大厨 协同工作，完成数据处理和分析报告

## 适用场景
- 业务数据统计和分析
- 数据可视化和报表生成
- SQL查询和数据提取
- 用户行为分析
- A/B测试结果分析
- 数据治理和质量检查

## 工作流
```
用户需求 → Data大厨（数据清洗、统计分析、可视化）
          → Code大厨（脚本开发、自动化处理、性能优化）
          → Doc大厨（报告生成、结论总结、建议提出）
          → 用户确认
```

## 使用示例
### 示例1：业务数据分析
**用户输入**：
> 这是上月的销售数据CSV，帮我做完整的销售分析，生成分析报告

**执行过程**：
1. Data大厨自动识别数据结构、清洗脏数据、做统计分析
2. 自动生成销售趋势图、地区分布图、产品占比图、客户画像
3. Code大厨开发自动化分析脚本，支持后续每月自动运行
4. Doc大厨整理成完整的数据分析报告，包含结论和业务建议

### 示例2：SQL查询和数据提取
**用户输入**：
> 帮我查询近30天的用户活跃数据，按地区和用户分层统计

**执行过程**：
1. Data大厨自动生成SQL查询语句，校验语法正确性
2. 执行查询，提取数据，做初步统计分析
3. 生成可视化图表，分析用户活跃趋势和分层结构
4. 输出完整的数据分析报告和可复用的SQL脚本

### 示例3：A/B测试结果分析
**用户输入**：
> 这是A/B测试的两组数据，帮我分析哪个版本效果更好，是否显著

**执行过程**：
1. Data大厨做显著性检验、置信度分析、效果对比
2. 分析核心指标差异、统计显著性、改进建议
3. Code大厨开发A/B测试分析模板，支持后续测试自动分析
4. 输出A/B测试分析报告，给出明确的结论和建议

## 常用指令
```
# CSV数据分析
@Data大厨 分析这个CSV文件，生成完整的分析报告

# SQL查询
@Data大厨 帮我写一个SQL查询，统计近30天的活跃用户，按地区分组

# 数据可视化
@Data大厨 基于这些数据，生成销售趋势图和地区分布图

# A/B测试分析
@Data大厨 分析这两组A/B测试数据，给出结论和建议

# 数据质量检查
@Data大厨 帮我检查这个数据集的质量，找出脏数据和异常值
```

## 配置说明
- 默认支持的数据格式：CSV、Excel、JSON、SQLite、MySQL、PostgreSQL
- 默认可视化库：Matplotlib、Seaborn、Plotly
- 默认统计方法：描述性统计、显著性检验、相关性分析
- 可以在Data大厨的配置文件中自定义数据源连接、分析模板、可视化样式
