# 项目管理套餐
> PM大厨 + BA大厨 + Doc大厨 协同工作，完成项目全流程管理

## 适用场景
- 新项目启动和规划
- 项目迭代管理
- 项目进度跟踪和风险预警
- 项目报告和文档生成
- 项目复盘和总结

## 工作流
```
用户需求 → PM大厨（项目规划、任务拆解）
          → BA大厨（需求分析、可行性研究）
          → Doc大厨（文档生成、格式优化）
          → 用户确认
```

## 使用示例
### 示例1：新项目规划
**用户输入**：
> 我要启动一个企业级AI助手项目，周期3个月，预算50万，帮我做完整的项目规划

**执行过程**：
1. PM大厨自动生成项目章程、WBS任务拆解、进度计划、资源分配、风险评估
2. BA大厨补充需求分析、可行性研究、ROI测算、效益评估
3. Doc大厨整理成完整的项目规划文档，包含甘特图、资源表、风险清单
4. 输出完整的项目规划方案，可直接用于评审

### 示例2：迭代进度跟踪
**用户输入**：
> 这是本周的项目进度数据，帮我做进度分析和风险预警

**执行过程**：
1. PM大厨分析进度偏差、资源使用情况、关键路径
2. BA大厨识别风险点、提出改进建议、调整后续计划
3. Doc大厨生成项目周报，包含进度仪表盘、风险预警、下周计划

### 示例3：项目复盘
**用户输入**：
> 项目上线了，帮我做完整的项目复盘报告

**执行过程**：
1. PM大厨分析项目目标达成情况、进度偏差、成本偏差、质量情况
2. BA大厨总结经验教训、改进措施、后续优化建议
3. Doc大厨生成项目复盘报告，包含数据分析、经验总结、改进计划

## 常用指令
```
# 新项目规划
@PM大厨 帮我做一个[项目名称]的完整规划，周期[X]个月，预算[X]万

# 任务拆解
@PM大厨 把这个需求拆解成具体的开发任务，分配到人，排好时间

# 风险评估
@PM大厨 帮我评估这个项目的风险点，给出应对措施

# 项目周报
@PM大厨 基于这些进度数据，生成本周的项目周报

# 项目复盘
@PM大厨 项目结束了，帮我做完整的复盘报告
```

## 配置说明
- 默认项目管理方法论：PMP BOK
- 默认迭代周期：2周
- 默认报告格式：Markdown + 飞书文档
- 可以在PM大厨的配置文件中自定义项目模板、报告格式、审批流程
