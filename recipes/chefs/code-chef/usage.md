# Code大厨 使用指南
## 安装方法
### 方法1：一键部署（推荐）
```bash
# 在ccg根目录运行
./scripts/deploy.sh --only code-chef
```

### 方法2：手动安装
```bash
# 安装智能体
openclaw agents install code-chef --config recipes/chefs/code-chef/config.yaml

# 验证安装
openclaw agents list | grep code-chef
```

## 配置自定义
### 1. 修改默认语言优先级
编辑 `config.yaml` 中的 `capabilities.languages` 字段，调整你常用的语言优先级：
```yaml
capabilities:
  languages:
    - "Python"  # 最常用的放最前面
    - "JavaScript/TypeScript"
    - "Go"
    - "Shell"
```

### 2. 自定义命令白名单
编辑 `config.yaml` 中的 `permissions.allowed_commands` 字段，添加你允许执行的命令：
```yaml
permissions:
  allowed_commands:
    - "python"
    - "node"
    - "your-custom-command"  # 添加你的自定义命令
```

### 3. 自定义代码模板
编辑 `config.yaml` 中的 `code_templates` 字段，添加你常用的代码模板：
```yaml
code_templates:
  your_custom_template: |
    # 你的自定义代码模板
    def hello():
        print("Hello World")
```

## 使用技巧
### 技巧1：快速生成脚本
```
@Code大厨 帮我写一个Python脚本：
- 功能：批量重命名目录下的所有文件，把文件名中的空格替换为下划线
- 输入：目录路径
- 输出：重命名统计报告
```
Code大厨会自动生成完整的脚本，包含错误处理和使用说明。

### 技巧2：代码审查
```
@Code大厨 帮我审查这段代码：
[粘贴你的代码]
- 重点检查：安全问题、性能问题、编码规范
- 给出具体的修复建议和代码示例
```
Code大厨会分级别列出问题，给出修复示例。

### 技巧3：Bug修复
```
@Code大厨 这段代码报错：
错误信息：[粘贴错误信息]
代码：[粘贴代码]
帮我定位问题并给出修复方案
```
Code大厨会快速定位问题原因，给出修复后的代码。

### 技巧4：技术选型咨询
```
@Code大厨 我要做一个高并发的API服务：
- 预期QPS：10000+
- 数据量：1000万+
- 技术栈：你推荐
- 请给出架构设计和技术选型建议
```
Code大厨会给出完整的技术选型方案，包括优劣势对比。

## 常见问题
### Q：Code大厨可以执行代码吗？
A：默认配置了命令白名单，只有在白名单中的命令可以执行，高风险命令（如删除、格式化）默认禁止。你可以在配置文件中自定义命令白名单。

### Q：生成的代码有Bug怎么办？
A：Code大厨生成的代码是基础版本，建议你先测试再使用，复杂功能可以分步骤开发，逐步验证。

### Q：可以支持我自己的私有技术栈吗？
A：可以，在系统提示词中添加你的技术栈信息和编码规范，Code大厨会按照你的要求生成代码。

### Q：可以生成单元测试吗？
A：可以，在指令中明确要求生成单元测试，Code大厨会自动生成对应的测试用例。

## 最佳实践
1. **需求明确**：生成代码前尽量明确需求、技术栈、编码规范，避免反复修改
2. **小步迭代**：复杂功能拆分成多个小任务，分步骤实现，逐步测试验证
3. **安全检查**：所有生成的代码尤其是涉及数据库、网络、文件操作的代码，一定要做安全检查
4. **代码复用**：常用的工具脚本和代码模板可以添加到配置中，后续直接复用
5. **代码审查**：重要功能的代码一定要经过人工审查，不要直接用于生产环境
