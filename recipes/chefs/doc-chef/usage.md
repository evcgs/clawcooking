# Doc大厨 使用指南
## 安装方法
### 方法1：一键部署（推荐）
```bash
# 在ccg根目录运行
./scripts/deploy.sh --only doc-chef
```

### 方法2：手动安装
```bash
# 安装智能体
openclaw agents install doc-chef --config recipes/chefs/doc-chef/config.yaml

# 验证安装
openclaw agents list | grep doc-chef
```

## 配置自定义
### 1. 新增文档模板
编辑 `config.yaml` 中的 `document_templates` 字段，添加你常用的文档模板：
```yaml
document_templates:
  your_custom_template: |
    # 你的自定义模板标题
    ## 第一部分
    ## 第二部分
    ## 第三部分
```

### 2. 调整支持的文档格式
编辑 `config.yaml` 中的 `capabilities.document.supported_formats` 字段，添加你需要的格式：
```yaml
capabilities:
  document:
    supported_formats: ["md", "docx", "pdf", "txt", "your-custom-format"]
```

### 3. 配置飞书应用权限
如果你有专用的Doc飞书应用，在飞书渠道配置中添加：
```yaml
feishu:
  sub_apps:
    doc:
      app_id: "cli_a92e72365878dcca"
      app_secret: "your-doc-secret"
```
Doc大厨会自动使用专用的飞书应用处理文档操作，避免和主应用冲突。

## 使用技巧
### 技巧1：快速生成文档
```
@Doc大厨 帮我写一份[文档类型]：
- 主题：[文档主题]
- 包含要点：[要点1，要点2，要点3]
- 输出格式：[Markdown/Word/PPT]
```
Doc大厨会按照你要求的格式生成完整的文档。

### 技巧2：文档摘要生成
```
@Doc大厨 帮我总结这个文档的核心内容：
[粘贴文档内容/飞书文档链接]
- 摘要长度：[300字/1000字]
- 重点突出：[核心要点/行动项/风险点]
```
Doc大厨会自动提取核心内容，生成简洁的摘要。

### 技巧3：表格数据处理
```
@Doc大厨 处理这个表格：
[粘贴表格数据/多维表格链接]
- 统计维度：[按部门/按时间/按人员]
- 生成图表：[柱状图/折线图/饼图]
- 输出报告：[包含分析建议/仅统计数据]
```
Doc大厨会自动统计分析，生成统计报告和可视化图表。

### 技巧4：批量文档处理
```
@Doc大厨 批量处理这个目录下的所有文档：
[目录链接/文件列表]
- 操作：[格式转换/内容提取/标签添加]
- 输出目录：[指定输出位置]
```
Doc大厨会批量处理所有文档，输出处理结果和统计报告。

## 常见问题
### Q：Doc大厨可以处理多大的文件？
A：默认支持最大100MB的文件，超大文件可以分段处理，或者在配置文件中调整`max_file_size`参数。

### Q：可以对接其他云存储吗？
A：默认支持飞书云盘，后续可以扩展支持Google Drive、OneDrive、阿里云盘等，需要在配置中添加对应的权限。

### Q：生成的文档可以直接同步到飞书吗？
A：可以，Doc大厨会自动把生成的文档保存到你指定的飞书目录，也可以生成分享链接。

### Q：支持多人协作编辑吗？
A：支持，Doc大厨可以作为协作者加入文档，自动处理编辑建议和版本合并。

## 最佳实践
1. **模板先行**：把常用的文档模板提前配置好，生成文档时直接套用，效率更高
2. **明确要求**：生成文档时明确说明主题、要点、格式、字数要求，结果更符合预期
3. **分段处理**：超大文档可以分段处理，避免一次处理内容太多导致的效果下降
4. **定期整理**：定期让Doc大厨整理知识库，添加标签和分类，方便后续检索
5. **备份重要文档**：重要文档操作前自动备份，避免误操作导致数据丢失
