# PM大厨 使用指南
## 安装方法
### 方法1：一键部署（推荐）
```bash
# 在clawcook根目录运行
./scripts/deploy.sh --only pm-chef
```

### 方法2：手动安装
```bash
# 安装智能体
openclaw agents install pm-chef --config recipes/chefs/pm-chef/config.yaml

# 验证安装
openclaw agents list | grep pm-chef
```

## 配置自定义
### 1. 修改默认模型
编辑 `config.yaml` 中的 `model.model_id` 字段，选择你想要使用的模型：
```yaml
model:
  model_id: "gpt-4o"  # 改为你的模型ID
```

### 2. 调整项目模板
编辑 `config.yaml` 中的 `workflows` 字段，自定义项目文档模板：
```yaml
workflows:
  project_planning:
    output_template: |
      # 你的自定义项目模板
      ## 一、自定义部分1
      ## 二、自定义部分2
      ...
```

### 3. 调整报告时间
编辑 `config.yaml` 中的 `capabilities.iteration` 字段：
```yaml
capabilities:
  iteration:
    default_duration: 7  # 改为一周迭代
    weekly_report_day: "周四"  # 改为周四生成周报
```

## 使用技巧
### 技巧1：快速生成项目规划
```
@PM大厨 帮我做一个[项目名称]规划：
- 周期：X个月
- 预算：X万
- 团队：X人（产品X，开发X，测试X）
- 核心目标：XXX
```
PM大厨会自动生成完整的项目规划，包含WBS、进度计划、资源分配、风险评估。

### 技巧2：自动生成周报
```
@PM大厨 生成周报：
- 本周完成：[列出完成的任务]
- 下周计划：[列出计划的任务]
- 遇到的问题：[列出问题]
```
PM大厨会自动整理成规范的项目周报格式。

### 技巧3：风险评估
```
@PM大厨 评估这个项目的风险：
- 项目类型：[项目类型]
- 技术栈：[技术栈]
- 团队情况：[团队情况]
- 时间要求：[时间要求]
```
PM大厨会识别潜在风险，给出优先级和应对措施。

### 技巧4：任务拆解
```
@PM大厨 拆解这个需求：
- 需求描述：[详细需求]
- 交付时间：[截止时间]
- 团队配置：[相关人员]
```
PM大厨会拆解成具体的任务，分配到人，排好时间。

## 常见问题
### Q：PM大厨可以对接我的项目管理工具吗？
A：可以，在配置文件中添加对应的工具集成配置，支持Jira、Trello、飞书项目等。

### Q：可以自定义项目模板吗？
A：完全可以，修改配置文件中的`workflows.output_template`即可，支持任意Markdown格式。

### Q：PM大厨生成的甘特图可以导出吗？
A：可以，默认生成Mermaid格式的甘特图，支持复制到Markdown文档中直接渲染，也可以导出为图片。

### Q：可以同时管理多个项目吗？
A：可以，PM大厨支持多项目并行管理，每个项目的数据独立存储，不会混淆。

## 最佳实践
1. **项目启动阶段**：让PM大厨先做完整的项目规划，提前识别风险，避免后续走弯路
2. **每日站会**：每天让PM大厨整理团队进度，自动生成站会纪要
3. **每周固定时间**：让PM大厨自动生成项目周报，同步给所有相关方
4. **每月末**：让PM大厨做项目月度复盘，总结经验教训
5. **项目结束**：让PM大厨做完整的项目复盘报告，沉淀组织过程资产
