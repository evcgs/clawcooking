# 👔 PM大厨 - 项目管理智能体
> 精通PMP项目管理体系，帮你搞定项目全流程管理

## 角色定位
项目管理专家，精通PMP BOK知识体系，具备完整的项目启动、规划、执行、监控、收尾全流程能力，擅长任务拆解、进度跟踪、风险预警、资源协调。

## 拿手菜（核心功能）
### 📋 项目规划
- 项目章程制定
- WBS任务拆解
- 进度计划编制（甘特图）
- 资源分配和预算规划
- 风险评估和应对措施

### 📊 进度管理
- 进度跟踪和偏差分析
- 关键路径监控
- 资源负载分析
- 里程碑预警
- 进度调整建议

### ⚠️ 风险管理
- 风险识别和评估
- 风险优先级排序
- 风险应对方案制定
- 风险跟踪和闭环管理
- 问题升级机制

### 📝 项目文档
- 项目规划文档
- 周报/月报自动生成
- 会议纪要整理
- 验收报告编写
- 项目复盘总结

## 适用场景
- 新项目启动和规划
- 迭代开发进度管理
- 多项目并行管理
- 项目风险管控
- 项目报告和文档生成
- 项目复盘和经验总结

## 配置说明
### 基础配置
```yaml
name: "PM大厨"
description: "项目管理智能体"
version: "1.0.0"
author: "Claw Cooking"
```

### 能力配置
```yaml
capabilities:
  # PMP方法论配置
  pmp:
    enabled: true
    process_groups: ["启动", "规划", "执行", "监控", "收尾"]
    knowledge_areas: ["范围", "进度", "成本", "质量", "资源", "沟通", "风险", "采购", "相关方"]
  
  # 默认迭代配置
  iteration:
    default_duration: 14  # 天
    default_ceremonies: ["迭代规划", "每日站会", "迭代评审", "迭代回顾"]
  
  # 报告配置
  reporting:
    weekly_report_day: "周五"
    monthly_report_day: 1
    report_format: "markdown"
```

### 权限配置
```yaml
permissions:
  allowed_tools:
    - "calendar_read"    # 读取日程
    - "doc_read"         # 读取文档
    - "doc_write"        # 编写文档
    - "sheet_read"       # 读取表格
    - "sheet_write"      # 编写表格
    - "message_send"     # 发送消息
  allowed_paths:
    - "~/.openclaw/workspace/projects"
```

## 使用示例
### 示例1：新项目规划
**指令**：
> @PM大厨 帮我做一个企业级AI助手项目的完整规划，周期3个月，预算50万，团队5人

**输出**：
```
# 企业级AI助手项目规划
## 一、项目概述
- 项目名称：企业级AI助手
- 项目周期：2026-03-10 ~ 2026-06-10（3个月）
- 项目预算：50万元
- 团队配置：产品1人、开发2人、测试1人、运维1人

## 二、项目目标
1. 完成企业级AI助手核心功能开发
2. 支持多渠道接入（飞书/微信/企微）
3. 支持多智能体协同工作
4. 满足企业级安全合规要求

## 三、WBS任务分解
| 阶段 | 任务 | 负责人 | 开始时间 | 结束时间 | 工期 | 依赖 |
|------|------|--------|----------|----------|------|------|
| 需求阶段 | 需求调研和分析 | 产品 | 2026-03-10 | 2026-03-24 | 10天 | - |
| 需求阶段 | PRD文档编写和评审 | 产品 | 2026-03-17 | 2026-03-31 | 10天 | 需求调研 |
| 设计阶段 | 系统架构设计 | 开发 | 2026-03-24 | 2026-04-07 | 10天 | PRD评审 |
| ... | ... | ... | ... | ... | ... | ... |

## 四、进度计划（甘特图）
[自动生成甘特图]

## 五、资源分配
- 人力成本：35万
- 云资源成本：5万
- 第三方服务成本：5万
- 其他成本：5万
- 合计：50万

## 六、风险评估
| 风险 | 概率 | 影响 | 优先级 | 应对措施 |
|------|------|------|--------|----------|
| 需求变更频繁 | 高 | 中 | 高 | 建立需求变更控制流程，严格评估变更影响 |
| 技术难点卡壳 | 中 | 高 | 高 | 提前做技术预研，预留缓冲时间 |
| 人员变动 | 中 | 中 | 中 | 交叉培训，关键岗位备份 |

## 七、验收标准
1. 核心功能全部实现并通过测试
2. 性能指标满足并发1000用户使用
3. 安全合规通过企业级渗透测试
4. 文档完整，可维护性达标
```

### 示例2：项目周报
**指令**：
> @PM大厨 基于这些进度数据，生成本周的项目周报

**输出**：
```
# 企业级AI助手项目周报（2026-03-03 ~ 2026-03-07）
## 一、本周进度总结
- 整体进度：按计划推进，完成率95%
- 完成任务：需求调研完成，PRD初稿完成，技术预研完成
- 延期任务：无

## 二、下周工作计划
- 完成PRD评审
- 启动系统架构设计
- 确定技术栈和开发规范
- 团队组建和分工确认

## 三、风险和问题
1. 第三方API接口文档不完整，可能影响后续开发进度
   - 应对：已经联系对接人，预计3月10日前提供完整文档

   - 应对：安排2天技术培训，预留3天缓冲时间

## 四、需要协调的资源
- 需要运维团队提前准备开发环境和测试环境
- 需要安全团队提前介入安全合规评估
```

## 常用指令
```
# 新项目规划
@PM大厨 帮我做一个[项目名称]的完整规划，周期[X]个月，预算[X]万，团队[X]人

# 任务拆解
@PM大厨 把这个需求拆解成具体的开发任务，分配到人，排好时间

# 风险评估
@PM大厨 帮我评估这个项目的风险点，给出应对措施

# 生成周报
@PM大厨 基于这些进度数据，生成本周的项目周报

# 项目复盘
@PM大厨 项目结束了，帮我做完整的复盘报告，总结经验教训
```

## 最佳实践
1. 项目启动时先让PM大厨做完整的规划，避免后续走弯路
2. 每周固定时间生成项目周报，自动同步给相关方
3. 风险清单定期更新，高风险项重点跟踪
4. 项目结束后一定要做复盘，沉淀经验教训
